# ベンチマーク

nvecdの類似検索パイプラインの実測パフォーマンスベンチマークです。

## 計測環境

- **ハードウェア**: Apple M4 Max (arm64), 128GB ユニファイドメモリ
- **SIMD**: NEON (4-wide, 128-bit), 4xマルチアキュムレータ展開
- **ビルド**: Release (`-O2`), Clang 16
- **設定**: デフォルト (`distance_metric: cosine`, `sample_size: 10000`)
- **計測方法**: 15回試行の中央値、ウォーム状態（compact storage構築済み）
- **再現方法**: `./bin/similarity_benchmark --gtest_also_run_disabled_tests`

> **ハードウェアに関する注記**: Apple Siliconはユニファイドメモリを使用しており、一般的なサーバーのDDR4/DDR5より高帯域です。AVX2搭載のx86サーバー（8-wide, 256-bit）では、SIMDレジスタ幅が2倍のため、絶対レイテンシは**より低く**なる見込みです。スケーリング特性はプラットフォーム間で一貫しています。

## 検索レイテンシ

### エンドツーエンド クエリ (SIM, dim=128, top_k=10, cosine)

| データセットサイズ | SearchByIdVectors (SIM) | SearchByVector (SIMV) | キャッシュヒット |
|---|---|---|---|
| 1K ベクトル | 0.012ms | 0.012ms | 0.25us |
| 10K ベクトル | 0.12ms | 0.12ms | 0.25us |
| 50K ベクトル | 0.62ms | 0.64ms | 0.25us |
| 100K ベクトル | **1.03ms** | **1.05ms** | **0.25us** |

**主な知見:**

- 50Kベクトルまでサンプリングなしでサブミリ秒
- 100Kベクトル: クエリあたり約1ms（**スレッドあたり約970 QPS**）
- キャッシュヒットレイテンシ: 250ナノ秒（**400万ops/秒**）
- 100Kスケールでキャッシュによる**4,000倍**の高速化

## パイプライン内訳 (100Kベクトル, dim=128)

| コンポーネント | 時間 | E2E比率 |
|---|---|---|
| NEON cosine スキャン (brute-force) | 0.80ms | 78% |
| Min-heap top-k 選択 | 0.05ms | 5% |
| ロック取得 + インデックス検索 | 0.03ms | 3% |
| 結果文字列構築 (k=10) | 0.02ms | 2% |
| **合計 (SearchByIdVectors)** | **1.03ms** | **100%** |

スキャン（SIMD dot product）が78%を占めます。Min-heapアプローチにより100Kの結果文字列割り当てを回避し、オーバーヘッドを約0.22ms（partial_sort）から約0.05msに削減。

## SIMD性能

### 距離演算 (dim=128, 1呼び出しあたり)

| 演算 | スカラー | NEON (4x展開) | AVX2 (4x展開) |
|---|---|---|---|
| Dot product | 約45ns | **約8ns** | **約4ns** (推定) |
| L2 norm | 約40ns | **約7ns** | **約3.5ns** (推定) |
| L2 distance | 約50ns | **約9ns** | **約4.5ns** (推定) |

NEON: スカラー比5-6倍。AVX2（推定）: スカラー比10-12倍。

### 4xマルチアキュムレータ最適化

Dot productループで4つの独立SIMDアキュムレータを使用し、FMAレイテンシを隠蔽:

| 手法 | dim=128 | dim=768 |
|---|---|---|
| シングルアキュムレータ | 約14ns | 約85ns |
| **4xマルチアキュムレータ** | **約8ns** | **約48ns** |
| 改善率 | 1.75倍 | 1.77倍 |

## 最適化の変遷

### 累積改善 (100Kベクトル, dim=128, top_k=10)

| 最適化 | E2Eレイテンシ | 前回比 | 累積 |
|---|---|---|---|
| ベースライン（ハッシュマップ総当たり） | 5.27ms | - | 1.0倍 |
| Compact連続ストレージ | 2.83ms | 1.86倍 | 1.86倍 |
| Min-heap top-k（文字列割当なし） | 1.66ms | 1.71倍 | 3.17倍 |
| 4x SIMDマルチアキュムレータ | 1.25ms | 1.33倍 | 4.22倍 |
| 統合ストレージ（二重map廃止） | **1.03ms** | 1.21倍 | **5.12倍** |

**合計: ベースラインから5.12倍高速化。**

## 統合ストレージ vs 二重ストレージ

| 指標 | 二重 (vectors_ + compact) | 統合 (compactのみ) |
|---|---|---|
| メモリ (100K, dim=128) | 約98MB | **約49MB** (50%削減) |
| メモリ (1M, dim=128) | 約1.1GB | **約550MB** (50%削減) |
| SetVectorレイテンシ | O(1)ハッシュ挿入 + 無効化 | O(dim) memcpy (**約0.5us**) |
| 書込後の読取ブロック | **160ms** (Compact()再構築) | **0ms** (常時有効) |
| DeleteVector | O(1)削除 | O(1)トゥームストーン |

統合ストレージにより、書き込みごとに発生していた160msのCompact()ブロッキングを完全解消。

## キャッシュ性能

### 選択的無効化 vs 全クリア

| 指標 | 全クリア（旧） | 選択的（新） |
|---|---|---|
| 無効化範囲 | 全エントリ | 影響エントリのみ |
| 無効化時間 | O(n)クリア | O(k)/アイテム |
| 書込後のキャッシュ残存率 | 0% | **約95-99%** |
| 書込下の実効ヒット率 | 低い | **高い** |

逆引きインデックス（`item_id -> set<CacheKey>`）により、変更されたアイテムを参照するキャッシュエントリのみを無効化。

### キャッシュ検索レイテンシ

| 操作 | レイテンシ | Ops/秒 |
|---|---|---|
| キャッシュミス（キー未発見） | 42-83ns | 1,200-2,400万 |
| キャッシュヒット（解凍+返却） | 250ns | 400万 |

## 近似検索（サンプリング）

### 精度 vs 速度のトレードオフ

| sample_size | カバレッジ (1Mベクトル) | レイテンシ | Recall@10 (目安) |
|---|---|---|---|
| 0 (厳密) | 100% | 約11ms | 100% |
| 50,000 | 5% | 約0.6ms | 約95% |
| **10,000 (デフォルト)** | **1%** | **約0.12ms** | **約80-90%** |
| 5,000 | 0.5% | 約0.06ms | 約70-80% |

Reservoir sampling（アルゴリズムR）による均一ランダムカバレッジ。推薦ワークロードでは`sample_size=10000`が速度/品質の良いバランス。

### 厳密検索を使用すべき場合

設定で`sample_size: 0`を指定:
- データセットが20K未満（フルスキャンでも十分高速）
- 厳密なtop-k結果が必要（例: 重複排除）
- クエリあたり10ms以上のレイテンシ予算がある場合

## デフラグメント性能

`DeleteVector()`後、トゥームストーンスロットが蓄積。`Defragment()`はフラグメンテーションが25%を超えると自動実行。

| データセットサイズ | デフラグメント時間 (25%トゥームストーン) |
|---|---|
| 1K ベクトル | 0.04ms |
| 10K ベクトル | 0.33ms |
| 50K ベクトル | 1.69ms |
| 100K ベクトル | 3.39ms |

## メモリサイジング

### ベクトルストレージ

```
メモリ = ベクトル数 x 次元 x 4バイト (float32)
       + ベクトル数 x 4バイト (ノルム)
       + ベクトル数 x 約64バイト (ID文字列 + インデックスオーバーヘッド)
```

| ベクトル数 | dim=128 | dim=384 | dim=768 |
|---|---|---|---|
| 10K | 6MB | 16MB | 31MB |
| 100K | 55MB | 155MB | 308MB |
| 1M | **550MB** | **1.5GB** | **3.1GB** |

## ベンチマークの実行

```bash
# Releaseモードでビルド
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel

# 全ベンチマーク実行
./build/bin/similarity_benchmark --gtest_also_run_disabled_tests

# 特定ベンチマークの実行
./build/bin/similarity_benchmark --gtest_also_run_disabled_tests \
  --gtest_filter="*SearchByIdVectors*"
```

ベンチマークテストは`benchmark`ラベルでCIから除外。安全のため`DISABLED_`プレフィックス付き。
